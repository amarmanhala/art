# Azure Carousel Image Upload Flow

This project uploads carousel images directly from the React admin dashboard to Azure Blob Storage using a short-lived SAS URL generated by the Go backend.

## Why Use SAS Uploads

The frontend should not know the Azure storage account key. The Go backend keeps the storage key private and creates a temporary upload URL with limited permissions.

Flow:

```txt
React admin -> Go API: request SAS upload URL
Go API -> Azure SDK: create write-only SAS URL
Go API -> React admin: return upload_url, image_url, blob_name
React admin -> Azure Blob Storage: PUT file directly using SAS URL
React admin: store/use returned image_url
```

## Azure Setup

1. Open the Azure Storage Account.
2. Go to **Security + networking > Access keys**.
3. Copy:
   - Storage account name
   - Key 1 value
4. Go to **Data storage > Containers**.
5. Copy the container name.
6. Add these backend env vars:

```env
AZURE_STORAGE_ACCOUNT_NAME=artbackendmedia
AZURE_STORAGE_ACCOUNT_KEY=your_key_value
AZURE_STORAGE_CONTAINER=carousel-images
AZURE_STORAGE_PUBLIC_URL=https://artbackendmedia.blob.core.windows.net/carousel-images
```

7. Configure **Settings > Resource sharing (CORS)** for Blob service:

```txt
Allowed origins:
http://localhost:5173
http://localhost:5174
http://localhost:5175

Allowed methods:
PUT, GET, HEAD, OPTIONS

Allowed headers:
*

Exposed headers:
*

Max age:
3600
```

## Backend Flow

Endpoint:

```txt
POST /api/uploads/carousel/sas
```

Request:

```json
{
  "file_name": "hero.png",
  "content_type": "image/png"
}
```

Backend does:

1. Validates Azure env vars.
2. Validates image content type.
3. Creates a unique blob path like:

```txt
carousel/abc123.png
```

4. Creates a SAS URL with:
   - HTTPS only
   - `Create` permission
   - `Write` permission
   - 10 minute expiry
5. Returns:

```json
{
  "upload_url": "https://...?...sas...",
  "blob_name": "carousel/abc123.png",
  "image_url": "https://artbackendmedia.blob.core.windows.net/carousel-images/carousel/abc123.png",
  "expires_at": "2026-06-13T20:36:44Z"
}
```

## Frontend Flow

The admin Crousel page does:

1. User selects image file.
2. React shows local thumbnail preview with `URL.createObjectURL(file)`.
3. User clicks **Save images**.
4. React requests SAS URL from Go API.
5. React uploads directly to Azure:

```http
PUT upload_url
Content-Type: image/png
x-ms-blob-type: BlockBlob
x-ms-blob-content-type: image/png
```

6. Azure stores the file.
7. React receives/keeps the final `image_url`.

## Important Notes

- Never expose `AZURE_STORAGE_ACCOUNT_KEY` in React.
- SAS URLs should be short-lived.
- The current flow uploads images and keeps the Azure URL in frontend state.
- The next step is saving carousel records in the database.

Suggested DB fields:

```txt
id
title
description
image_url
blob_name
sort_order
is_active
created_at
updated_at
```

## Common Errors

`AZURE_NOT_CONFIGURED`

The backend env vars are missing or the backend was not restarted.

Browser CORS error

Azure Blob CORS is missing `PUT` or the frontend origin.

Azure 403

The SAS URL is expired, generated with wrong permissions, or created for the wrong container/key.

Image uploaded but not visible

The container may be private. Enable blob read access or serve images through backend/read SAS URLs.
